Git là gì?

Git là hệ thống quản lý phiên bản phân tán (DVCS) — mỗi developer có toàn bộ lịch sử project ngay trên máy mình. Không cần internet để commit, không sợ mất code.

📸

Snapshot, không phải Delta

Git lưu toàn bộ trạng thái file (snapshot) ở mỗi commit, không phải chỉ lưu sự thay đổi. Điều này giúp checkout nhanh và đáng tin cậy.

🌐

Phân tán (Distributed)

Mỗi bản clone là một repo đầy đủ với toàn bộ history. Không phụ thuộc server trung tâm để làm việc hàng ngày.

🔐

Hash SHA-1

Mỗi commit được định danh bởi hash SHA-1 duy nhất. Nội dung thay đổi → hash thay đổi → dữ liệu không thể bị giả mạo ngầm.

⚡

Branching siêu nhanh

Tạo branch trong Git chỉ là tạo một file nhỏ (40 bytes). Không cần copy cả project như SVN cũ.

🗂️

3 Khu vực chính

Working Directory — nơi bạn edit file.
Staging Area — chuẩn bị cho commit.
Repository — lịch sử lưu mãi mãi.

🔗

Linked List of Commits

Mỗi commit trỏ về commit cha. Tạo thành một chuỗi lịch sử có thể duyệt ngược về bất kỳ thời điểm nào.

3 trạng thái của file trong Git
Modified
File đã thay đổi trong Working Directory nhưng chưa được staged.
Staged
File đã được đánh dấu sẽ vào commit tiếp theo (đã git add).
Committed
Dữ liệu đã được lưu an toàn trong database local của Git.
Untracked
File mới tạo, Git chưa theo dõi. Cần git add để bắt đầu track.

Branch & Merge

Branch là siêu năng lực của Git. Tạo, merge, rebase — xem hình ảnh trực quan từng thao tác.

🌿

git merge

Gộp lịch sử của 2 branch lại. Tạo thêm 1 merge commit. Lịch sử được giữ nguyên, rõ ràng thời điểm branch nào join vào main.

✏️

git rebase

Viết lại lịch sử — di chuyển commit sang base mới. Lịch sử linear, sạch đẹp hơn nhưng KHÔNG dùng với branch đã share với người khác.

🍒

git cherry-pick

Lấy đúng 1 commit cụ thể từ branch khác áp lên branch hiện tại. Hữu ích khi chỉ muốn lấy 1 fix mà không merge cả branch.

🔀

Fast-forward vs 3-way merge

Nếu main không có commit mới sau khi tạo feature branch → fast-forward (chỉ dời pointer). Ngược lại → 3-way merge tạo merge commit.

Git Log & History

Đọc và hiểu lịch sử commit — kỹ năng thiết yếu để debug, review, và rollback.

Ví dụ git log --oneline --graph --all
$ git diff HEAD~1..HEAD — app.js
function greet(name) {
- return 'Hello ' + name;
+ return `Hello, ${name}!`;
+ // thêm dấu phẩy và exclamation
}
🔍

git log --oneline --graph

Xem lịch sử dạng ASCII graph. Thấy toàn bộ cây branch/merge một cách trực quan ngay trong terminal.

🔎

git blame

Xem ai đã viết từng dòng code và commit nào. Không phải để "blame" — mà để hiểu context của code.

🔬

git bisect

Binary search qua lịch sử commit để tìm commit nào gây ra bug. Cực kỳ mạnh khi debug regression.

📦

git reflog

"Bảo hiểm" của Git. Lưu mọi thứ HEAD đã trỏ đến — dù reset --hard hay xóa branch vẫn có thể khôi phục.
